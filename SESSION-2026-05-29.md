### Bug 2: Missing choice buttons on events
- Symptom: Event text appeared (in narrative log) but no buttons, no overlay
- Root cause: My override script replaced button onclick with `submitChoice(idx, ch)` — a function that doesn't exist
- The engine uses `game.chooseEventChoice(i)` with double-click guard via `btn.dataset.clicked`
- Present event text appeared in narrative via `say()` but overlay never opened because `presentEvent()` threw before adding `active` class
- Additional issue: wrapper pattern (call original → add fallback) was fragile — if original threw, wrapper code never ran
- **Final fix:** Removed wrapper entirely. Patched original `presentEvent` to:
  - Log event ID and choice count to console
  - Detect if `ev.choices` is not an array
  - Add red **Continue** fallback button always appended to choices container

## Technical Lessons Learned

1. **Node.js is broken** on this machine — segfaults with large files. Can't use `new Function()` or `vm` on the full engine file. Use Python harness or manual browser testing.

2. **Browser tool limitations:** The headless browser tool renders pages empty for large HTML files with inline scripts. This is a tool limitation, not a file issue. Always verify on a real device.

3. **Override pattern is fragile:** Wrapping engine functions by replacing `window.presentEvent` is dangerous — if original throws, wrapper code never executes. Better to patch the original function inline.

4. **`game` object scope:** The `game` object is created inside `createGame()` closure, not globally accessible as `window.game`. Can't access it from console after page load in browser tool.
